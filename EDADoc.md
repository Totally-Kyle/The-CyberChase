# Explain the data transformations, which the team will likely implement based on your EDA, Although you may have already agreed to several transformations

Following our exploratory data analysis (EDA) of the February 7th and 8th honeypot logs, several key data transformations are being implemented to enhance interpretability, reduce redundancy, and prepare the dataset for machine learning. A major step involves geotagging the Source and Destination IP addresses using the MaxMind GeoLite2 database. This produced two new fields—Source Location and Destination Location—which offer human-readable geographic context to better understand where attacks originate and where they are directed. With these in place, the original IP fields will be dropped for simplicity. Categorical variables such as Label, Protocol, and the new location fields will be one-hot encoded to convert them into numerical form suitable for machine learning models.

To address the dataset's high dimensionality (83 features), we’re pursuing a structured reduction strategy. We began by conducting a greater-than-zero analysis to flag inactive or low-variance features. We then used a correlation heatmap to highlight highly correlated features, which often add redundancy rather than insight. To go further, we grouped features by functional similarity—such as packet length metrics, flow statistics, and inter-arrival times—and created grouped histograms and group-specific heatmaps. This revealed not only overlapping information but also patterns of skewness or concentration that influence model design decisions. Based on these visual and statistical findings, we are now determining which features to retain, combine, or drop within each group—allowing us to reduce the feature space thoughtfully while preserving the dataset’s predictive power.
