# Explain the data transformations, which the team will likely implement based on your EDA, Although you may have already agreed to several transformations

After combining the honeypot logs from February 7th and 8th, I added a date column to keep track of which day each record came from. This makes it easier to look at everything together while still being able to compare activity across the two days. I also geotagged the Source and Destination IPs using the MaxMind GeoLite2 database, which added two new location columns that show where each request came from and where it was sent. These human-readable fields offer a clearer view of global activity, so I’ll be dropping the raw IP addresses to keep things clean. Categorical columns like Protocol, Label, and the new location fields will eventually be one-hot encoded to prep the data for machine learning.

To better understand the structure of the data, I grouped features by type—such as packet stats, flow metrics, TCP flags, and timing information—and created histograms and correlation heatmaps for each group. This helped surface patterns like skewed distributions or strong correlations that could impact model design. While I'm still piecing together the full story, one standout so far is that Protocol 17 was by far the most common during this period, and Iran appeared as the top source of traffic. I’ll continue digging into patterns like time-of-day behavior and other context as I refine which features to keep for modeling.
